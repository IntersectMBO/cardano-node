  • ncExperimentalProtocolsEnabled :: !Bool

    During the development and integration of new network protocols (node-to-node and node-to-client) we wish to be able to test them but not have everybody use them by default on the mainnet. Avoiding enabling them by default makes it practical to include such not-yet-ready protocol versions into released versions of the node without the danger that node operators on the mainnet will start using them prematurely, before the testing is complete.

    The flag defaults to False

    This flag should be set to True when testing the new protocol versions.

  • ncMempoolTimeoutSoft :: DiffTime

    If the mempool takes longer than this to validate a tx, then it discards the tx instead of adding it.

  • ncMempoolTimeoutHard :: DiffTime

    If the mempool takes longer than this to validate a tx, then it disconnects from the peer.

    WARNING: if this is less than mempoolTimeoutSoft, then mempoolTimeoutSoft is irrelevant. If it's equal or just barely larger, then the soft/hard distinction will likely be unreliable.

  • ncMempoolTimeoutCapacity :: DiffTime

    If the mempool takes longer than this cumulatively to validate when each entered the mempool, then the mempool is at capacity, ie it's full, ie no tx can be added.

data ResponderCoreAffinityPolicy Source #

We expose the ForkPolicy as a NodeConfiguration field.

  • NoResponderCoreAffinity corresponds to noBindForkPolicy
  • ResponderCoreAffinity corresponds to responderForkPolicy with a randomIO generated salt and getNumCapabilities.
