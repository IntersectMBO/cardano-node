data NetworkTopology adr Source #

Describes the non-P2P topology of a node. Whenever the node actually runs, the type parameter adr should be RemoteAddress. However, we might want to use and serialize this type with adr being NodeId, or another placeholder type, if we want the user to be able to edit the topology without knowing the actual addresses of the nodes: those might only be knowable at runtime.
