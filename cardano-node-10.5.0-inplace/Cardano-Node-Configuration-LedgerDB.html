data LedgerDbSelectorFlag Source #

Choose the LedgerDB Backend

As of UTxO-HD, the LedgerDB now uses either an in-memory backend or LMDB to keep track of differences in the UTxO set.

  • V2InMemory: uses more memory than the minimum requirements but is somewhat faster.
  • V1LMDB: uses less memory but is somewhat slower.
  • V1InMemory: Not intended for production. It is an in-memory reproduction of the LMDB implementation.
